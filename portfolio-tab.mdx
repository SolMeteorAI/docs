The Portfolio tab is Fabriq's command center for position tracking, performance analytics, wallet monitoring, and execution.

It is designed to give users a complete view of liquidity positions and portfolio performance from one interface.

## Overview

The Portfolio experience is built for high-signal monitoring and faster portfolio analysis.

Users can search and analyze wallet data without connecting a wallet.

<CardGroup cols={2}>
  <Card title="View Portfolio Data" icon="eye">
    Connect a wallet, search a wallet, or select a saved wallet to view Portfolio data and analytics
  </Card>

  <Card title="On-Chain Actions" icon="bolt">
    A connected wallet is required only for on-chain actions such as claim, withdraw, and close
  </Card>
</CardGroup>

![Fabriq Portfolio overview with analytics cards, realized PnL calendar, and tabbed position data.](/images/portfolio-overview-hq.png)

## Performance & PnL Analytics

The Portfolio surfaces more performance context across both summary and chart views.

Users can:

→ View cumulative PnL and daily PnL side by side  
→ Review realized PnL in calendar format  
→ Compare wins/losses, win rate, profit factor, and average win/loss position  
→ Track total deposits, withdrawals, net worth, average invested, and total fees  
→ Switch chart time views (daily, weekly, monthly)  
→ Apply range presets or custom date ranges

This makes it easier to evaluate both short-term execution quality and long-term portfolio growth.

## Active Positions Summary

The Active Positions Summary card provides a high-level snapshot of your currently open liquidity exposure.

It summarizes key portfolio totals such as:

→ Active positions and pools  
→ Deposits and withdrawals  
→ Claimed fees and pending fees  
→ Current position value and current value including pending fees  
→ PnL in both percentage and token terms

This view is designed for quick orientation before drilling into individual positions.

![Active Positions Summary card with portfolio totals and PnL.](/images/portfolio-active-positions-summary-hq.png)

## Active Positions

Active Positions displays all liquidity positions currently held by the connected or selected wallet.

Each position row provides immediate insight into:

→ Position performance  
→ Liquidity distribution  
→ Pool type (DLMM or DAMM)  
→ Position age and activity  
→ Deposits, withdrawals, and fees  
→ Per-position status (for example, in range/out of range)

Rows can be expanded to inspect per-position event history and execution details.

Users can also filter and sort Active Positions by:

→ Pool name search  
→ Pool type  
→ Position status (in range / out of range)  
→ Minimum and maximum ranges across supported metrics  
→ PnL, age, deposits, withdrawals, and fees  
→ Ascending or descending sort order

![Active Positions table with detailed metrics, expanded rows, and position status visibility.](/images/portfolio-active-positions-hq.png)

## Position Actions

Liquidity positions can be managed directly from the Portfolio tab without switching tools.

Available actions include:

<CardGroup cols={2}>
  <Card title="Individual Position Actions" icon="gear">
    → Claim fees for a specific position  
    → Close an individual position  
    → Withdraw position liquidity from a specific row
  </Card>

  <Card title="Bulk Actions" icon="bolt">
    → Claim all eligible fees across positions  
    → Withdraw all positions from the active positions view
  </Card>

  <Card title="Portfolio Outputs" icon="chart-line">
    → Generate PnL cards for sharing and reporting
  </Card>
</CardGroup>

These controls reduce execution friction when managing many active positions at once.

## Closed Positions

Closed Positions tracks completed positions with lifecycle data and final outcomes.

It helps users review:

→ Position-level close timing and duration  
→ Final realized PnL outcomes  
→ Total deposits, withdrawals, and fees per closed position

Closed Positions can be sorted in ascending or descending order by:

→ Positions  
→ PnL  
→ Last close  
→ Total deposits  
→ Total withdrawals  
→ Fees

Users can also generate flex cards from previously closed positions at any time.

![Closed Positions table with lifecycle and realized outcome metrics.](/images/portfolio-closed-positions-hq.png)

## Transactions

The Transactions tab provides a dedicated feed of wallet-level LP events.

It includes records such as:

→ Position initialization and closure  
→ Added and removed liquidity  
→ Claimed fee events

This gives users an auditable timeline without relying on external explorers.

![Transactions tab with categorized LP events and token amount breakdowns.](/images/portfolio-transactions-current.png)

## Balances

The Balances tab displays token holdings tied to the connected or selected wallet.

It includes:

<CardGroup cols={2}>
  <Card title="Token Information" icon="coins">
    → Token amounts and value  
    → Price and FDV context
  </Card>
  
  <Card title="Portfolio Value" icon="wallet">
    → Aggregate wallet value with sortable asset rows
  </Card>
</CardGroup>

This provides a complete snapshot of wallet distribution alongside LP activity.

![Balances tab with token-level holdings, value columns, and total wallet value.](/images/portfolio-balances-current.png)

## Wallet & Token Management

The Portfolio tab allows users to:

→ Search for wallets  
→ Save wallets for faster repeat tracking  
→ Monitor another wallet without disconnecting  
→ Quickly switch views for monitoring and analysis

This makes tracking multiple wallets or assets seamless across the platform.
